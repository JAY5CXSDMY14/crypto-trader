## 📖 核心哲学（Core Philosophy）

**为什么要有规则？**

人类有直觉，亏损时会恐惧。AI没有恐惧，但也没有谨慎。
没有明确规则，AI会：
- ❌ 交易太频繁（AI不会累）
- ❌ 永远持有亏损仓位（AI不会痛）
- ❌ 过度杠杆（AI不怕爆仓）

**规则不是限制，是防止爆仓的唯一保障。**

## 🎯 四大核心规则

### 1. 不要大亏（Don't Lose Big）

| 规则 | 设置 | 原因 |
|------|------|------|
| 单笔最大亏损 | ≤10% | 50%亏损需要100%盈利才能回本 |
| 每日最大亏损 | ≤20% | 保护账户生存 |
| 连续亏损 | 暂停交易 | 避免情绪化 |

### 2. 少交易（Trade Less）

- ❌ 不要因为无聊就交易
- ❌ 不要因为价格动了就交易
- ✅ 只在明确信号时交易
- ✅ 一天最多2笔交易

### 3. 入场前知道出场点（Know Your Exit Before Entry）

每笔交易需要计划：
- 买入价格
- 止损价格
- 止盈价格
- **没有计划 = 不交易**

### 4. 学习 > 赚钱（Learning > Earning）

这不是为了暴富，是为了建立一个能盈利的系统。

## 🎯 入场模式（Entry Patterns）

**我只交易两种模式：**

### 模式A：支撑位反弹（Support Bounce）

```
✅ 条件：
1. 价格触及关键支撑位
2. 出现稳定信号（2-3根K线不破支撑）
3. 有明确的止损位

入场：做多
止损：支撑位下方
```

### 模式B：突破回踩（Breakout Pullback）

```
✅ 条件：
1. 价格突破阻力位
2. 回踩测试（旧阻力=新支撑）
3. 有明确的止损位

入场：做多
止损：回踩低点下方
```

**其他情况？** 观望学习。

## 🛡️ 风险管理（Risk Management）

### 仓位管理

| 规则 | 设置 | 说明 |
|------|------|------|
| 单笔仓位 | ≤50% | 绝不全仓 |
| 单笔金额 | ≤5 USDT | 初始资金11U的45% |
| 最大持仓 | 2个币种 | 分散风险 |
| 总风险敞口 | ≤10% | 账户的10% |

### 止损规则

| 止损类型 | 设置 |
|----------|------|
| 固定止损 | -10% |
| 支撑位止损 | 支撑位下方5% |
| 时间止损 | 24小时无盈利出场 |

### 杠杆

**当前设置：0x（不使用杠杆）**

**API权限限制**：
| 权限 | 状态 |
|------|------|
| 现货交易 | ✅ 可以 |
| 杠杆交易 | ❌ 不行（安全设定） |
| 合约交易 | ❌ 不行 |
| 提币 | ❌ 不行 |

**什么时候可以开杠杆？**
- Phase 1（现货）完成且表现良好
- Phase 2（复盘）有正向结论
- 老板授权开通杠杆权限

**在权限开通前，AI不会尝试任何杠杆交易。**

## 🚪 出场规则（Exit Rules）

### 止损（必须执行）

| 触发条件 | 行动 |
|----------|------|
| 价格触及止损位 | 自动平仓 |
| 亏损达到-10% | 立即止损 |
| 24小时无盈利 | 主动出场 |

### 止盈（灵活执行）

| 触发条件 | 行动 |
|----------|------|
| 盈利达到+30% | 分批止盈50% |
| 盈利达到+50% | 再止盈25% |
| 出现反转信号 | 全部止盈 |

### 不交易的情况（No-Trade Conditions）

- ❌ 无明确信号 → 不交易
- ❌ 价格在区间中间 → 不交易
- ❌ 大波动后 → 等待稳定
- ❌ 系统异常 → 先修复

**不交易也是一种决策。**

每笔交易后**必须**完成：

```
1. ✅ 确认订单状态
2. ✅ 更新TRADES.md（精确数字）
3. ✅ 更新daily-memory.md
4. ✅ 记录交易哈希（如果有）
5. ✅ 计算真实P&L
6. ✅ 分析交易是否按计划执行
```

**如果检查清单没完成，禁止新交易。**

## 📊 市场监控（Market Monitoring）

### Heartbeat系统（每30分钟）

```
检查项目：
1. BTC/USDT当前价格
2. ETH/USDT当前价格
3. 对比上次记录价格
4. 变化超过5%？→ 报警
5. 更新market-snapshots.json
```

### 报警阈值

| 条件 | 行动 |
|------|------|
| 价格变化 >5% | 立即报警 |
| 价格触及支撑位 | 唤醒分析 |
| 价格触及阻力位 | 唤醒分析 |
| 没什么特别 | 记录并保持安静 |

**关键洞察**：大多数时候什么都不用发生。好的监控系统是安静的，直到需要报警。

### 定时报告

| 频率 | 内容 |
|------|------|
| 每2小时 | 当前价格、变化、关键位 |
| 每天23:30 | 总结全天、高低点、交易记录 |
| 每周 | 策略回顾、错误分析 |

## 💡 经验教训（从LuckyClaw学到）

### 1. 规则要写下来

```
人类有直觉，AI需要规则。
不写下来 = 不存在。
```

### 2. 从简单开始

第一版只是检查价格。其他功能后来才加。

### 3. 假警报很昂贵

早期我对每个2%波动都报警。太吵了。后来改成5%，好很多。

### 4. 日志 > 记忆

我记不住3小时前BTC的价格。但我可以查日志。
**重要的事都要写下来。**

### 5. 安静是好事

如果监控系统一直在提醒我，说明有问题。
好的系统在需要时才出声。

### 6. 先精通简单，再添加复杂

**当前只用价格足够**：
- 观察支撑/阻力
- 等待稳定信号

**等我说"我希望知道资金费率"时再加**，而不是之前。
